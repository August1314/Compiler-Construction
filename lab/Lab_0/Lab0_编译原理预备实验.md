1. 实验目标

本实验面向 Java 语言的初学者，假设学生已掌握结构化程序设计方法，对面向
对象程序设计方法有初步了解。学生最好至少已掌握一门 Java 之外的面向对象
程序设计语言，例如 C++或 Smalltalk-80。

序设计方法在建模与设计上的主要区别，为后续的编译原理实验打下初步基础。

3. 实验内容

3.1 基本要求
设计一个基于命令行界面的计算个人所得税（工资、薪金所得）应用程序，根据

用户输入的当月工资薪金总额计算应缴纳的个人所得税额。为适应未来个人所得

税法的可能变化（历史上曾有不同城市执行不同标准的情况，也曾发生过立法修

改个人所得税起征基数），该程序应支持对个人所得税起征点进行调整的功能，

并且个人所得税各级税率也允许修改。请使用简单的命令行菜单方式让用户选择

所需的功能。

3.2 个人所得税计算方法
我国税法规定，公民每月工资、薪金收入总额超过 5,000 元者应缴纳个人所得税，

即个人所得税起征点，2006 年修订立法改为 1,600 元，2011 年调至 3,500 元。

收入总额减去免征基数后剩余部分称为应纳税所得额，应纳税所得额按下表规定

的超额累进税率计算应缴纳的个人所得税。实际上，个人所得税税率表共分 7 级，

3.3 测试用例
例如，张三某月工资薪金总收入为 50,000 元，减去免征基数 5,000 元，则应纳税
所得额为 45,000 元。应纳税所得额中，超过 35000 部分为 45000–35000=  10000
元，按 30%税率计算应纳税 3000 元；而 25000-35000 元的部分为 10000 元，按
25%税率计算应纳税 2500 元；12000-25000 元部分为 13000 元，按 20%税率计算
应纳税 2600 元；3000-12000 元部分为 9000 元，按 10%税率计算应纳税 900 元；
0-3000 元部分为 3000 元，按 3%税率计算应纳税 90 元。所以，该月张三应缴纳
的个人所得税金额为 3000+2500+2600+900+90=9090 元。

5.  评分标准

编程风格：
1.  源程序中标识符命名、程序版面、程序注释等影响程序可理解性的要素（需

要遵循 Sun 公司建议的 Java 编码规范）。

3.  程序对未来变化的适应性如何，譬如是否容易修改或不必修改就应对可能发
生的变化，是否方便在不同的环境下（例如一个基于 J2EE 的 Web 应用中）
