1. 实验目标

在原始实验软装置基础上，实现可提交、可复现、可测试的后缀表达式转换程序，满足评分项中功能、规范性与鲁棒性要求。

  1. 将 Parser.lookahead 从静态成员改为实例成员。
  2. 消除 rest() 尾递归，改为循环实现。
  3. 扩展错误处理：错误分类、错误定位、错误信息输出，并支持 --recover。
  4. 补全 Javadoc 文档注释并生成 HTML 文档。
  5. 完成 JUnit 单元测试与递归/迭代性能比较。

3. 输入语言定义

输入为一行中缀表达式，仅允许数字 0~9 与运算符 +、-，不允许空白字符。

Expr  -> Term (('+'|'-') Term)*
Term  -> digit
digit -> '0'..'9'

程序输出对应后缀表达式；当输入有误时，stdout 追加 (error)，并在 stderr 输出详细错误信息。

4.1 尾递归消除

原程序通过 rest() 递归处理后续运算。新版本以 expectOperand 和 pendingOp 维护状态，使用循环完成等价逻辑，减少递归调用开销。

语法错误缺运算符、缺运算量、空表达式95+2 / 9-5+-2SYNTAX error at position n

11. 实验结论

  1. 将 lookahead 由静态改为实例字段后，状态管理更符合面向对象设计。
  2. 尾递归改循环后，控制流更直接，便于扩展与维护。
  3. 错误分类与定位显著提升了程序鲁棒性和可诊断性。
  4. 跨平台脚本使提交结果在 macOS 与 Windows 上均可复现。

12. 实验心得与反思

12.1 从功能正确到工程可交付

本实验最大的提升不在“能否输出后缀”，而在于把解析器、错误处理、测试、文档和脚本打通，形成可验证、可复现的完整提交物。

12.2 设计取舍

为了兼容样例输出，stdout 保持简洁；详细错误输出放在 stderr，使自动比对与问题定位互不干扰。

12.3 后续改进

  • 支持括号、乘除与优先级。
  • 完善错误恢复策略（同步符号/短语级恢复）。
  • 接入 CI 自动执行编译、测试与文档构建。
