本实验要求在既有实验软装置中补全 parser.Calculator 的实现，使其能够正确处理复杂表达式的求值，并在出现错误输入时抛出与测试框架一致的异常类型。

  • 支持算术、关系、逻辑、三目运算和预定义函数调用。
  • 支持整数、浮点数与科学计数法。
  • 支持布尔常量与数值 / 布尔双类型约束。
  • 保持实现简单直接，避免超出实验需求的过度设计。

3. 设计与实现

整体结构

  • Lexer：负责生成 Token。
  • Token / TokenType：描述词法单元。
  • Value：承载数值型或布尔型中间结果。
  • 递归下降解析函数：按优先级逐层求值。

设计取舍

  • 不额外构造 AST，直接在解析过程中求值。
  • 用手写数字扫描控制非法小数格式。
  • 在 requireNumber() / requireBoolean() 中集中处理类型错误。

关键实现点

  • 通过 parseConditional() 将三目运算作为最低优先级入口，并做分支类型一致性检查。
  • 通过 parseRelation() 把关系运算结果统一转换成布尔值，避免后续算术层误用。
  • 通过 invokeUnaryMathFunction() 与 invokeExtremaFunction() 统一封装函数参数检查逻辑。

6. 关键代码片段

双类型值对象

private static final class Value {
    private final boolean booleanType;
    private final double numberValue;
    private final boolean booleanValue;

    private double requireNumber()
        throws TypeMismatchedException {
        if (booleanType) {
            throw new TypeMismatchedException(
                "Numeric value expected.");
        }
        return numberValue;
    }
}

三目运算处理

private Value parseConditional()
    throws ExpressionException {
    Value condition = parseLogicalOr();
    if (lexer.peek().type != TokenType.QUESTION) {
        return condition;
    }
    lexer.next();
    boolean branch = condition.requireBoolean();
    Value trueValue = parseConditionalOperand();
    expect(TokenType.COLON, ...);
    Value falseValue = parseConditionalOperand();
    if (trueValue.booleanType != falseValue.booleanType) {
        throw new TypeMismatchedException(...);
    }
    return branch ? trueValue : falseValue;
}
